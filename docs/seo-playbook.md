## 7. AI Search Optimization (GEO -- Generative Engine Optimization)

### 7.1 Google AI Overviews

To appear in Google's AI Overview for queries like "best thai restaurant gatineau":

**What helps:**
- Strong E-E-A-T signals (Experience, Expertise, Authoritativeness, Trustworthiness)
- FAQ schema (already implemented)
- Restaurant schema with aggregateRating (already implemented)
- Content that directly answers questions in 50-70 word summaries
- Consistent citations across 30+ sources (builds entity authority)
- Fresh weekly GBP posts and recent reviews

**Specific actions:**
1. Ensure blog posts open with a concise direct answer (AI loves extractable summaries)
2. Use question-phrased H2 headings that match voice search queries
3. Include factual, verifiable data (prices, hours, ratings)
4. Add `speakable` schema markup to key content sections

### 7.2 ChatGPT Optimization

ChatGPT pulls from Bing's index and web sources. To appear in ChatGPT when users ask "best thai food in gatineau":

1. **Ensure Bing indexing** (Section 2.2) -- ChatGPT search uses Bing
2. **Build citations everywhere** -- ChatGPT references TripAdvisor, Yelp, food blogs
3. **Get mentioned on third-party review sites** -- The more places your brand appears with consistent information, the more likely AI cites you
4. **Keep content up to date** -- AI systems deprioritize stale content
5. **Create content in conversational Q&A format** -- ChatGPT prefers content that directly addresses user questions

**Test regularly:** Ask ChatGPT "What are the best Thai restaurants in Gatineau?" and "Can you recommend a Cambodian restaurant near Ottawa?" -- monitor if Krong Thai appears.

### 7.3 Perplexity Optimization

Perplexity heavily weights:
- **Citation frequency** (~35% of ranking factors) -- being mentioned across many sources
- **Source authority** -- high-DA sites like TripAdvisor, Yelp, news outlets
- **Structured data** -- FAQ schema, Restaurant schema, Menu schema
- **Content freshness** -- regularly updated content

**Actions:**
1. Get listed on as many directories as possible (Section 3)
2. Get featured in food blog articles and local news (Section 5)
3. Ensure structured data validates without errors
4. Update website content regularly (blog posts, menu changes, seasonal items)

### 7.4 Structured Data Additions for AI

The site already has:
- Restaurant schema with aggregateRating
- Menu schema
- FAQ schema
- BreadcrumbList schema

**Add these for better AI visibility:**

1. **Speakable schema** -- tells AI which sections are suitable for voice/text extraction:
```json
{
  "@type": "WebPage",
  "speakable": {
    "@type": "SpeakableSpecification",
    "cssSelector": [".hero-description", ".faq-answer", ".restaurant-description"]
  }
}
```

2. **ItemList schema for menu sections** -- helps AI understand menu structure:
```json
{
  "@type": "ItemList",
  "name": "Signature Dishes",
  "itemListElement": [
    { "@type": "MenuItem", "name": "Bangkok Soup", "description": "...", "offers": { "price": "X.XX" } }
  ]
}
```

3. **Review schema with individual reviews** -- supplement the existing aggregateRating:
```json
{
  "@type": "Review",
  "author": { "@type": "Person", "name": "..." },
  "reviewRating": { "ratingValue": "5" },
  "reviewBody": "..."
}
```

### 7.5 Content Format AI Prefers

- **Direct answers** in first sentence (AI extracts these as snippets)
- **Bullet points and numbered lists** (AI can parse structured content)
- **Tables** comparing dishes, prices, or options
- **FAQ format** with schema markup
- **Statistics and data points** (e.g., "93% approval rating", "since 1999", "220+ reviews")
- **Consistent entity mentions** (always "Krong Thai Restaurant" not "the restaurant")

## 9. Technical SEO Remaining

### 9.1 Core Web Vitals Targets (2025-2026)

| Metric | Target | What It Measures |
|--------|--------|-----------------|
| LCP (Largest Contentful Paint) | < 2.5 seconds | How fast the main content loads |
| INP (Interaction to Next Paint) | < 200 milliseconds | How fast the page responds to user input |
| CLS (Cumulative Layout Shift) | < 0.1 | Visual stability during loading |

**How to monitor:**
- Google Search Console > "Core Web Vitals" report
- [PageSpeed Insights](https://pagespeed.web.dev/) -- test every page
- [web.dev/measure](https://web.dev/measure/) -- detailed audit
- Chrome DevTools > Lighthouse tab

**Common issues for Astro sites:**
- Font loading causing CLS -- the site loads Playfair Display and DM Sans from Google Fonts. Add `font-display: swap` (likely already set) and consider self-hosting fonts for faster LCP
- Hero images -- ensure the hero images use `loading="eager"` and are properly sized with `width` and `height` attributes
- Third-party scripts -- minimize external script loading

### 9.2 Mobile-First Indexing Checklist

- [x] Viewport meta tag present
- [x] Responsive design (Tailwind CSS)
- [ ] Test all pages with [Google's Mobile-Friendly Test](https://search.google.com/test/mobile-friendly)
- [ ] Ensure touch targets are at least 48x48 CSS pixels with 8px spacing (site targets 44x44 -- consider upgrading)
- [ ] Base font size at least 16px
- [ ] No horizontal scrolling on any page
- [ ] All content visible without JavaScript (Astro SSG handles this)
- [ ] Test forms on mobile (contact and catering forms)

### 9.3 Additional Technical Improvements

**1. Self-host fonts (performance improvement):**
Currently loading from Google Fonts CDN (line 65 of Head.astro). Self-hosting eliminates DNS lookup, connection, and download from external origin.


**2. Add WebP fallbacks for all images:**

**3. Image `alt` text audit:**

**4. Internal linking strategy:**

**5. Page speed optimizations:**

- Minimize CSS with Tailwind's purge (handled by Vite)
